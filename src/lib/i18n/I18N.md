# Internationalization (EN/FR)

This site uses `react-i18next` with bundled JSON under `src/locales/{en,fr}/`.

## Rules for new copy
1. Do **not** hardcode user-facing UI strings in components.
2. Add the English key to `src/locales/en/<namespace>.json`.
3. Add the same key to `src/locales/fr/<namespace>.json` (draft FR is fine pending review).
4. Read it with `useTranslation("<namespace>")` and `t("key.path")`.
5. Run `npm run i18n:check` (also runs in `npm run build`) — missing FR keys fail the build.
6. **Do not translate** proper nouns: project names, brand/partner names, people names, or place names (e.g. “Valley Park Library”, “Toronto, ON”). Keep those in the source data as English.

## Namespaces
- `common` — nav, footer, cookies, intro splash, sector nav labels
- `home` — homepage sections
- `about` — About Us page
- `contact` — Contact page + inquiry/quote form
- `services` — Services index chrome, category/card labels, detail chrome, default process
- `serviceDetails` — per-slug overview / bestFor / deliverables / FAQs
- `sectors` — Sectors index, market hubs (`/markets/[slug]`), market/sub-sector card copy
- `sectorDetails` — deep `/sectors/[slug]` page bodies
- `portfolio` — Portfolio index + project detail chrome/scopes/specs (names & locations stay EN)
- `careers` — Careers page + application form
- `insight` — Insight / LinkedIn newsroom page (`/linkedin`)
- `legal` — Legal Centre chrome + per-doc titles (full policy bodies still placeholders)

Later phases will add store namespaces. Capability PDF statements remain EN for now. Full legal document bodies should be counsel-owned when they replace placeholders.
